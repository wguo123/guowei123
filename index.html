Team Data Science Process Tutorial

This tutorial provides instructions on how to set up Team Data Science Process (TDSP) environment for different roles in the organization.

Group Manager

  1. Set up group VSTS server: weig-ds.visualstudio.com
    • Sign up VSTS server
    • Create Project Template Repo (empty), named R1
    • Create Utility Repo (empty), named R2
  2. Seed Project Template (R1) and Utilities (R2) from TDSP server (from public GitHub in the future) as Group-Project-Template and Group-Utilities
    • Clone two repos (TDSP Project Template Repo, named R01, and TDSP Utilities, named R02) from TDSP server (GitHub), as D01 and D02
    • Clone R1 and R2 as D1 and D2 (directories on local machine)
    • Copy D01 to D1, D02 to D2
    • Make changes to D1 and D2, if needed
    • Push D1 and D2 back to R1 and R2 on group server
  3. Add group members to group server

Team Lead

  1. Assumptions: R1 and R2 have been set up by group lead
  2. Create team repo in VSTS server: ds-team-1
    • Rename the default repository as Team-1-Project-Template (R3)
    • Create a new empty repo under team repo: Team-1-Utilities (R4)
  3. Copy Group Project Template and Group Utilities to team repo as Team-1-Project-Template and Team-1-Utilities
    • Clone Group-Project-Template R1 and Group-Utilities R2 from group VSTS server to local machine, as D1 and D2
    • Clone R3 and R4 to local machine as D3 and D4
    • Copy D1 to D3, and copy D2 to D4
    • Make changes to D3 and D4, if needed
    • Push D3 and D4 to R3 and R4 on group server
  4. Create shared storage (file share, Team FS) for team resources: Team-1-share-drive (Optional)
  5. Mount file share (Team FS) (Optional)
  6. Add team members

Project Lead

  1. Assumption: R3 and R4 have been set up by team lead
  2. Create project-1 template repo (R5) under Team project
  3. Seed Project-1-template repo R5
    • Clone Team-1-Project-Template R3 and Team-1-Utilities R4 from team repo to local machine as D3 and D4
    • Clone Project-1-Template repo R5 to local machine as D5
    • Copy D3 to D5
    • Make changes to D5 if needed
    • Push D5 to Project-1-Template repo R5
  4. Create shared storage or project resource: Project-1-share-drive (FS Project) (Optional)
  5. Mount FS-Team and FS-Project to local machine (Optional)
  6. Add project members

Project IC

  1. Assumption: R2, R4, and R5 have been created and seeded by team lead and project lead
  2. Clone R5 and R4 to local machine as D5 and D4
  3. Clone R2 to local machine as D2 (Optional)
  4. Mount project shared storage: Project-1-share-drive (Optional)
  5. Mount team shared storage: Team-1-share-drive (Optional)

Tools

  2. Create SSH public key
  3. Provision Data Science Virtual Machine
